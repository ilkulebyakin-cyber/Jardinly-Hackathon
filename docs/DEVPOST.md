## Feedback Session ID

The majority of Build Week work was performed in one long Codex session. In the
Codex composer, run `/feedback`, choose to include/share the existing session,
and submit it. Paste the session ID returned by that dialog into Devpost.

Local session identifier for the working thread:

`019f7baa-6b80-7ba1-82fb-97577c4d1e71`

Use the ID returned after `/feedback` if it differs; submitting feedback is the
step that makes the session available to the event reviewers.
